راهنمای عملیاتی مدیر سیستم: گام ۲ (اقتصاد سوخت و پایداری)
================================================================================
نسخه: ۱.۰
وضعیت: حیاتی (Critical)

رفیق، برای اینکه سیستم مالی و لجر (Ledger) رهنما فعال بشه، باید تغییرات زیر رو دقیقاً در پنل سوپابیس اعمال کنی. این تغییرات زیرساخت "دفتر کل" رو می‌سازن.

گام ۱: اجرای اسکریپت SQL (ارتقای تابع کسر سوخت)
-----------------------------------------------
۱. به بخش SQL Editor در پنل سوپابیس برو.
۲. یک کوئری جدید باز کن.
۳. محتویات فایل زیر رو کپی و اجرا کن:
   مسیر: /supabase/code/sql/step2_fuel_economy_upgrade.sql

   *توضیح فنی:* این اسکریپت تابع 'deduct_fuel' رو بازنویسی می‌کنه تا هر بار که شارژ کم میشه، یه رکورد منفی توی جدول 'reward_ledger' هم ثبت کنه. همچنین پالیسی‌های امنیتی (RLS) رو برای دیدن تاریخچه فعال می‌کنه.

گام ۲: بررسی ایندکس‌ها (Performance Check)
------------------------------------------
بعد از اجرای اسکریپت بالا، برو به بخش Table Editor > reward_ledger.
مطمئن شو که ایندکس 'idx_reward_ledger_created_at' ساخته شده باشه. این برای اینکه لیست تراکنش‌ها توی موبایل سریع لود بشه، حیاتیه.

گام ۳: کانفیگ دستی (اختیاری)
----------------------------
ما فعلاً پکیج‌های خرید (۱ ساعت، ۵ ساعت، ۱۰ ساعت) رو توی کد فرانت‌اِند (TopUpModal.tsx) ثابت گذاشتیم.
اگر خواستی قیمت‌ها رو عوض کنی، نیازی به دیتابیس نیست، مستقیم توی اون فایل تغییر بده.

نکته امنیتی نهایی:
مطمئن شو که جدول 'reward_ledger' هم توی لیست Replication (بخش Realtime) باشه تا اگر تراکنشی ثبت شد، لیست کاربر آنی آپدیت بشه. (البته این معمولاً از قبل فعاله).

تست صحت عملکرد:
یک بار توی اپلیکیشن دکمه "شارژ سوخت" رو بزن و یه پکیج الکی بخر. باید صدای سکه بیاد و توی "دفتر کل سوخت" یه رکورد جدید سبز رنگ ببینی.
